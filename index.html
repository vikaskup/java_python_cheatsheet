Useful imports (paste these first)

When & whyOn LeetCode the heavy lifters live in libraries — pull them in before you start typing so you never stall mid-solution. In Python, four imports cover ~90% of problems: collections (deque for BFS, defaultdict/Counter for grouping & counts), heapq (priority queues / Dijkstra / top-K), bisect (binary search on sorted data), and functools.lru_cache (instant top-down DP). In Java a single import java.util.*; exposes every collection you need. Keep this list as your blank-editor starting point.

I/O & program skeleton

On LeetCode you fill in a method, but knowing the full skeleton helps for OA / Codeforces-style judges.

When & whyLeetCode hands you a pre-written method signature, so you rarely touch I/O there. But online assessments (HackerRank, Codeforces, company OAs) make you read from stdin and print results yourself — and a slow reader is the #1 cause of "time limit exceeded" on large inputs. Learn the fast-input idiom once: in Java a single Scanner is fine up to ~10⁵ tokens but use BufferedReader beyond that; in Python always rebind input = sys.stdin.readline the moment input size exceeds a few thousand lines.

JAVA
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        String s = sc.next();          // one token
        sc.nextLine();                 // consume newline
        String line = sc.nextLine();   // whole line
        System.out.println(n + " " + s);
    }
}
PYTHON
import sys
input = sys.stdin.readline   # faster input

def main():
    n = int(input())
    s = input().split()[0]       # one token
    line = input().rstrip("\n")  # whole line
    print(n, s)

main()

Tip: In Java, mixing nextInt() then nextLine() leaves a dangling newline — call an extra nextLine(). In Python, default input() is slow for big inputs; rebinding to sys.stdin.readline is the classic speed fix.

Types & variables

Java is statically typed; Python is dynamic. Watch integer ranges.

When & whyThe single most important thing to internalize when moving between these languages for interviews is integer range. Java's int caps at ~2.1×10⁹; any sum, product, or hash that can exceed that must be a long, or you get silent wrong answers that are brutal to debug. Python integers grow without bound, so problems that are "tricky" in Java (factorials, big powers, prefix-sum overflow) are non-issues. The flip side: Python's flexibility costs speed, so for tight time limits Java's fixed-width primitives and arrays are genuinely faster.

JAVA
int a = 5;            // 32-bit, ~2.1e9 max
long big = 10_000_000_000L;   // use long for big sums
double d = 3.14;
char c = 'a';
boolean flag = true;
String s = "hi";
int[] arr = new int[n];       // defaults to 0
var list = new ArrayList<Integer>();  // local type inference
PYTHON
a = 5                 # arbitrary precision int (no overflow)
big = 10_000_000_000
d = 3.14
c = 'a'               # just a length-1 string
flag = True
s = "hi"

Gotcha: Java int overflows silently at ~2.1×10⁹ — for sums/products use long. Python ints never overflow, so a problem that's "hard" in Java because of overflow is trivial in Python.

Numbers & math

When & whyYou reach for these constantly: min/max in DP transitions, abs for distances, bit ops for subsets and parity, and "infinity" sentinels to initialize best-so-far values. The two traps that sink interviews are division/modulo on negatives (the languages disagree — see the tip below) and choosing the right "infinity": use Integer.MAX_VALUE/float('inf') so that dist[u] + w comparisons behave, but in Java guard against overflow when you add to MAX_VALUE. Bit shifts (1 << k) are the backbone of subset enumeration and bitmask DP.

JAVA
Math.max(a, b); Math.min(a, b);
Math.abs(x); Math.pow(2, 10);  // returns double
(int) Math.sqrt(n);
Math.floorDiv(-7, 2);   // -4 (floor)
-7 / 2;                 // -3 (truncates toward 0)
-7 % 2;                 // -1 (sign follows dividend)
Math.floorMod(-7, 2);   // 1  (always non-negative)
Integer.MAX_VALUE; Integer.MIN_VALUE;
Long.MAX_VALUE;
Integer.bitCount(x);    // popcount
1 << k; x >> 1; x & 1;
PYTHON
max(a, b); min(a, b)
abs(x); 2 ** 10          # exact int
int(n ** 0.5)            # or math.isqrt(n)
-7 // 2                  # -4 (floor division)
-7 / 2                   # -3.5 (true division -> float)
-7 % 2                   # 1  (sign follows divisor)

float('inf'); float('-inf')

1 << k; x >> 1; x & 1

Key difference — division & modulo: Python // and % floor toward negative infinity (so -7 % 2 == 1). Java / and % truncate toward zero (so -7 % 2 == -1). For wrap-around indices with negatives, use Python % directly, but in Java use Math.floorMod. For "infinity" sentinels use Integer.MAX_VALUE / float('inf').

Strings

Both are immutable. Build with StringBuilder / list-join for O(n).

When & whyStrings are immutable in both languages, which has one massive performance implication: concatenating in a loop with + is O(n²) because each + copies the whole string so far. Any time you build output character-by-character — reversing, decoding, joining tokens — accumulate into a StringBuilder (Java) or a list then "".join() (Python). Use this section's slicing and search methods for parsing problems, and remember a string is essentially a read-only char array you can index and compare.

Characters

When & whyCharacter arithmetic is the trick that makes string problems fast. Mapping a letter to an index with c - 'a' lets you use a tiny fixed-size int[26] array instead of a hash map — faster, and it makes anagram/frequency comparisons a single array equality check. Use the isDigit/isLetter family for tokenizers and "valid palindrome / parse expression" problems. Note the conceptual gap: Java has a true char primitive (an integer code unit), while in Python a "character" is just a length-1 string, so you go through ord()/chr() instead of implicit int conversion.

JAVA
char c = 'a';
int v = c - 'a';          // 0..25 index
char back = (char)('a' + v);
Character.isDigit(c);
Character.isLetter(c);
Character.isLetterOrDigit(c);
Character.isUpperCase(c);
Character.toLowerCase(c);
int d = c - '0';          // char digit -> int
int[] freq = new int[26];
freq[c - 'a']++;
PYTHON
c = 'a'
v = ord(c) - ord('a')     # 0..25 index
back = chr(ord('a') + v)
c.isdigit()
c.isalpha()
c.isalnum()
c.isupper()
c.lower()
d = int(c)                # or ord(c) - ord('0')
freq = [0] * 26
freq[ord(c) - ord('a')] += 1

Tip: The c - 'a' / ord(c) - ord('a') trick to map letters to 0–25 array indices is one of the most-used patterns in string problems (anagrams, frequency counts).

Lists / Dynamic arrays

When & whyThis is your default container — a resizable, index-addressable sequence backed by a contiguous array, so random access and append-at-end are O(1) (amortized), but inserting/removing at the front or middle is O(n) because everything shifts. Reach for it whenever order matters and you mostly index or append. If you find yourself repeatedly removing from the front, that's a signal to switch to a deque/queue. Java distinguishes a fixed primitive int[] (fastest, known size) from a growable ArrayList<Integer> (boxed, flexible); Python's list is one type that does both jobs.

2D arrays / grids

When & whyGrids appear in a huge fraction of problems — islands, maze/shortest-path, matrix DP, game boards. The reusable muscle memory here is the 4-direction (or 8-direction) neighbor loop plus an in-bounds check; you'll write it dozens of times, so memorize the DIRS array pattern. Pair it with a visited matrix (or mutate the grid in place) for BFS/DFS traversals. The dimensions convention grid[row][col] with R=len(grid), C=len(grid[0]) is near-universal — keep it consistent to avoid transposition bugs.

JAVA
int[][] g = new int[rows][cols];   // zero-filled
g[r][c] = 1;
int R = g.length, C = g[0].length;
int[][] dp = new int[n][m];
boolean[][] seen = new boolean[R][C];

// 4-directional neighbors:
int[][] DIRS = {{0,1},{0,-1},{1,0},{-1,0}};
for (int[] d : DIRS) {
    int nr = r + d[0], nc = c + d[1];
    if (nr>=0 && nr<R && nc>=0 && nc<C) { /* ... */ }
}
// ragged copy:
int[][] copy = new int[R][];
for (int i=0;i<R;i++) copy[i]=g[i].clone();
PYTHON
g = [[0]*cols for _ in range(rows)]   # zero-filled
g[r][c] = 1
R, C = len(g), len(g[0])
dp = [[0]*m for _ in range(n)]
seen = [[False]*C for _ in range(R)]

# 4-directional neighbors:
DIRS = [(0,1),(0,-1),(1,0),(-1,0)]
for dr, dc in DIRS:
    nr, nc = r + dr, c + dc
    if 0 <= nr < R and 0 <= nc < C:
        pass  # ...

# deep-ish copy:
copy = [row[:] for row in g]

Critical Python gotcha: Never write [[0]*C]*R — that makes R references to the same inner list, so editing one row edits all. Always use the comprehension [[0]*C for _ in range(R)]. Chained comparison 0 <= nr < R is a clean Python-only bounds check.

HashMap / Dictionary

When & whyThe hash map is the workhorse that turns O(n²) brute force into O(n): any time the question is "have I seen this value / what's its count / where did it last appear," a map gives O(1) average lookup. Two Sum, group-anagrams, longest-substring, prefix-sum-equals-k — all hinge on it. Use a plain HashMap/dict when you only need membership and counts. Step up to a TreeMap/sorted structure only when you also need ordered iteration or floor/ceiling queries (e.g. "nearest smaller key"), since that costs O(log n) per op instead of O(1).

JAVA
Map<String,Integer> m = new HashMap<>();
m.put("a", 1);
m.get("a");                  // null if absent
m.getOrDefault("a", 0);
m.containsKey("a");
m.merge("a", 1, Integer::sum);   // counter idiom
m.put("a", m.getOrDefault("a",0)+1);
m.remove("a");

for (int v : m.values()) {}
m.computeIfAbsent(k, x -> new ArrayList<>()).add(v);
// ordered:
TreeMap<Integer,Integer> tm = new TreeMap<>();
tm.firstKey(); tm.lastKey();
tm.floorKey(x); tm.ceilingKey(x);

Tip: Java's merge(k, 1, Integer::sum) and computeIfAbsent mirror Python's Counter and defaultdict. Need sorted keys / floor-ceiling lookups? Java has TreeMap built in; in Python reach for sortedcontainers.SortedDict (available on LeetCode) or bisect.

HashSet / Set

When & whyA set is a hash map without values — use it purely for membership: "is this in the visited set," "are there duplicates," "what's the intersection of two groups." O(1) average add/contains. Classic uses: detect a cycle by tracking seen nodes, dedupe results, and the longest-consecutive-sequence trick (put everything in a set, then only start counting from numbers whose predecessor is absent). When you also need sorted order or floor/ceiling, switch to TreeSet (Java) or SortedList (Python's sortedcontainers).

s.removeAll(other);         // difference

Frequency counting / Counter

When & why"Counting how many of each" is its own micro-pattern that shows up in anagrams, "can you rearrange to form X," top-K frequent elements, and sliding-window-with-character-budget problems. For lowercase letters or a small fixed alphabet, an int[26] is both fastest and lets you compare two windows with one array-equality check. For arbitrary keys, Python's Counter bundles construction, most_common, and multiset arithmetic; Java leans on map.merge(k,1,Integer::sum). The interview insight: comparing whole frequency tables is how you check anagrams in O(n) instead of sorting in O(n log n).

JAVA
Map<Character,Integer> cnt = new HashMap<>();
for (char c : s.toCharArray())
    cnt.merge(c, 1, Integer::sum);

// fixed-alphabet fast path:
int[] freq = new int[26];
for (char c : s.toCharArray()) freq[c-'a']++;

// compare two frequency arrays (anagram):
Arrays.equals(freqA, freqB);
PYTHON
from collections import Counter
cnt = Counter(s)              # whole string at once
cnt['a']                      # 0 if absent (no KeyError)
cnt.most_common(2)            # top-2 (val, count)
cnt - Counter(t)              # multiset subtract
sum(cnt.values())

# anagram check in one line:
Counter(s) == Counter(t)

Tip: Python's Counter is a Swiss-army knife: construct from any iterable, most_common(), arithmetic between counters, and equality for anagram checks. Java has no direct equivalent — the int[26] frequency array is both the idiom and the faster choice for lowercase-letter problems.

Stack (LIFO)

When & whyReach for a stack whenever the problem has nesting or "most recent unmatched" structure: valid parentheses, evaluate an expression, decode nested strings, simplify a Unix path, or simulate function-call/backtracking iteratively. The mental cue is "I need to remember things and undo them in reverse order." It's also the engine behind the monotonic-stack pattern (its own section below). O(1) push/pop/peek.

JAVA
// Prefer Deque over the legacy Stack class:
Deque<Integer> st = new ArrayDeque<>();
st.push(1);     // add to head
st.push(2);
st.peek();      // 2  (top, null if empty)
st.pop();       // 2  (remove top)
st.isEmpty();
st.size();
PYTHON
# a plain list IS a stack:
st = []
st.append(1)    # push
st.append(2)
st[-1]          # peek top (IndexError if empty)
st.pop()        # remove & return top
not st          # is empty
len(st)

Tip: In Java avoid the old java.util.Stack (synchronized, slow) — use ArrayDeque as a stack. In Python a list's append/pop at the end are O(1) and serve as the canonical stack. Always guard pop()/peek() against empty.

Queue / Deque (FIFO & double-ended)

When & whyA queue processes things in arrival order — its defining use is BFS, where FIFO ordering is exactly what guarantees you explore the graph level by level (and therefore find shortest paths in unweighted graphs). A double-ended queue (deque) adds O(1) operations at both ends, which unlocks the sliding-window-maximum trick (monotonic deque) and any "add/remove from either side" simulation. The non-negotiable rule: for queue behavior use ArrayDeque / collections.deque — never list.pop(0), which is O(n).

JAVA
Deque<Integer> q = new ArrayDeque<>();
q.offer(1);        // add to tail (enqueue)
q.offerFirst(0);   // add to head
q.peek();          // head
q.peekLast();      // tail
q.poll();          // remove head (dequeue)
q.pollLast();      // remove tail
q.isEmpty();
q.size();
// classic BFS loop:
while (!q.isEmpty()) {
    int cur = q.poll();
    // process, then q.offer(next);
}
PYTHON
from collections import deque
q = deque()
q.append(1)        # add to tail (enqueue)
q.appendleft(0)    # add to head
q[0]               # head
q[-1]              # tail
q.popleft()        # remove head (dequeue) O(1)
q.pop()            # remove tail
not q              # is empty
len(q)
# classic BFS loop:
while q:
    cur = q.popleft()
    # process, then q.append(next)

Gotcha: Never use a Python list as a queue — list.pop(0) is O(n). Always use collections.deque for BFS. In Java, ArrayDeque covers both stack and queue roles; don't use LinkedList unless you need indexed list ops.

Heap / PriorityQueue

Both are min-heaps by default.

When & whyA heap gives you the smallest (or largest) element in O(1) and insert/extract in O(log n) — without keeping everything fully sorted. Reach for it whenever you repeatedly need "the current best": top-K problems (keep a size-K heap, O(n log k)), merge K sorted lists, the running median (two heaps), task scheduling, and Dijkstra/Prim where you always pull the closest frontier node. The decision rule vs. sorting: if you need the extreme element repeatedly as data changes, heap; if you need everything ordered once, just sort.

JAVA
PriorityQueue<Integer> pq = new PriorityQueue<>();   // min
pq.offer(3); pq.offer(1);
pq.peek();      // smallest
pq.poll();      // remove smallest
pq.size();

// MAX-heap:
PriorityQueue<Integer> mx =
    new PriorityQueue<>(Collections.reverseOrder());

// heap of pairs by 2nd field, then 1st:
PriorityQueue<int[]> h = new PriorityQueue<>(
    (a,b) -> a[1]!=b[1] ? a[1]-b[1] : a[0]-b[0]);
h.offer(new int[]{node, dist});
PYTHON
import heapq
pq = []
heapq.heappush(pq, 3); heapq.heappush(pq, 1)
pq[0]               # smallest (peek)
heapq.heappop(pq)   # remove smallest
len(pq)
heapq.heapify(arr)  # O(n) build in place

# MAX-heap: push negatives
heapq.heappush(pq, -x); -heapq.heappop(pq)

# heap of tuples: sorts by 1st, then 2nd, ...
heapq.heappush(pq, (dist, node))
d, node = heapq.heappop(pq)

# k largest / smallest helpers:
heapq.nlargest(k, nums); heapq.nsmallest(k, nums)

Tip: For a max-heap, Java takes a comparator (Collections.reverseOrder()); Python has no max-heap, so push -x and negate on pop. Python tuples in a heap compare element-by-element automatically — perfect for Dijkstra's (dist, node). In Java use int[] with a custom comparator.

Sorting & comparators

When & whySorting is the cheapest way to expose structure: it's the setup step for two-pointer, greedy-interval, and binary-search-on-data solutions, costing O(n log n). The skill that actually gets tested is writing a custom comparator — sort intervals by start, sort people by age then name, sort by frequency descending. Master the multi-key form (Java's thenComparing, Python's tuple key (a, b)) and the descending trick (negate a numeric key in Python; .reversed() in Java). Both sorts are stable, which matters when ties must preserve input order.

JAVA
Arrays.sort(arr);                 // primitives, asc
Collections.sort(list);
list.sort(Comparator.naturalOrder());
list.sort(Comparator.reverseOrder());

// by key:
people.sort(Comparator.comparingInt(p -> p.age));
// descending by key:
people.sort(Comparator.comparingInt((P p)->p.age).reversed());
// multi-key: age asc, then name asc:
people.sort(Comparator.comparingInt((P p)->p.age)
                      .thenComparing(p -> p.name));
// 2D array by column 0:
Arrays.sort(intervals, (a,b) -> a[0]-b[0]);
// safe for big ints (avoid overflow):
Arrays.sort(intervals, (a,b) -> Integer.compare(a[0],b[0]));
PYTHON
arr.sort()                        # in place, asc
arr.sort(reverse=True)            # desc

Number theory (GCD / LCM / primes / fast power)

When & whyA small toolbox covers most math-flavored problems. GCD (Euclid's algorithm) underlies fraction reduction, "is line through points," and water-jug puzzles; LCM derives from it. // Sieve of Eratosthenes: primes up to n
boolean[] sieve(int n) {
    boolean[] comp = new boolean[n + 1];
    for (int i = 2; (long)i * i <= n; i++)
        if (!comp[i])
            for (int j = i * i; j <= n; j += i)
                comp[j] = true;
    return comp;   // comp[k]==false && k>=2 -> prime
}

// Fast power: a^b mod m  in O(log b)
long power(long a, long b, long m) {
    long res = 1; a %= m;
    while (b > 0) {
        if ((b & 1) == 1) res = res * a % m;
        a = a * a % m;
        b >>= 1;
    }
    return res;
}

Matrix transforms (rotate / transpose / spiral)

When & whyThese are pattern-recognition problems: once you know the index mapping, the code is short. Transpose swaps m[i][j] with m[j][i]; rotating 90° clockwise is "transpose, then reverse each row" (counter-clockwise: reverse rows first, then transpose) — this lets you rotate in place. Spiral traversal maintains four shrinking boundaries (top/bottom/left/right). Set-matrix-zeroes uses the first row/column as marker storage for O(1) space. Recognize by "rotate the image," "print spiral," "diagonal traverse."

JAVA
// Rotate n x n 90° clockwise, IN PLACE:
void rotate(int[][] m) {
    int n = m.length;
    // 1) transpose
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++) {
            int t = m[i][j]; m[i][j] = m[j][i]; m[j][i] = t;
        }
    // 2) reverse each row
    for (int[] row : m)
        for (int l = 0, r = n - 1; l < r; l++, r--) {
            int t = row[l]; row[l] = row[r]; row[r] = t;
        }
}
// Spiral order:
List<Integer> spiral(int[][] m) {
    List<Integer> res = new ArrayList<>();
    int top=0, bot=m.length-1, left=0, right=m[0].length-1;
    while (top <= bot && left <= right) {
        for (int j=left;j<=right;j++) res.add(m[top][j]);
        top++;
        for (int i=top;i<=bot;i++) res.add(m[i][right]);
        right--;
        if (top<=bot) for (int j=right;j>=left;j--) res.add(m[bot][j]);
        bot--;
        if (left<=right) for (int i=bot;i>=top;i--) res.add(m[i][left]);
        left++;
    }
    return res;
}

When & whyUse binary search whenever the search space is monotonic — sorted data, or a yes/no predicate that flips exactly once (false…false true…true). It turns O(n) scans into O(log n). The lower_bound/upper_bound template (loop while lo < hi, return lo) handles insertion points and "first/last occurrence."

JAVA
// exact target -> index, else -1
int binarySearch(int[] a, int target) {
    int lo = 0, hi = a.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;   // avoid overflow
        if (a[mid] == target) return mid;
        else if (a[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}

        if (a[mid] < t) lo = mid + 1; else hi = mid;

Cyclic sort (numbers in range 1..n)

When & whyWhen an array contains n integers drawn from the range 1..n (or 0..n-1), each value has a natural "home" index, so you can sort in O(n) time and O(1) space by swapping each number to its correct slot. After the pass, the first index whose value is wrong reveals the missing/duplicate number. This is the go-to for "find the missing number," "find all duplicates," "find the first missing positive," and "set mismatch" — problems that look like they need a hash set but don't.

JAVA
// place each value v at index v-1 (range 1..n)
void cyclicSort(int[] a) {
    int i = 0;
    while (i < a.length) {
        int correct = a[i] - 1;          // home index of a[i]
        if (a[i] != a[correct]) {         // not in place -> swap
            int t = a[i]; a[i] = a[correct]; a[correct] = t;
        } else i++;
    }
}
// then: first index where a[i] != i+1 is the missing number
int firstMissing(int[] a) {
    cyclicSort(a);
    for (int i = 0; i < a.length; i++)
        if (a[i] != i + 1) return i + 1;
    return a.length + 1;
}
PYTHON

Sliding window

When & whySliding window is the specialization of two pointers for contiguous subarrays/substrings with a constraint: "longest substring with ≤ K distinct," "smallest subarray with sum ≥ target," "max sum of window size k." Both pointers only move forward, so each element is added and removed at most once → O(n). The mental model: grow the right edge greedily; when the window violates the constraint, shrink the left edge until it's valid again, tracking the best window seen. Fixed-size windows just slide both edges together. If the problem says "subarray/substring" + "longest/shortest/at most/exactly," this is almost always the tool.

JAVA
// longest substring with no repeats:
int lengthOfLongest(String s) {
    Map<Character,Integer> last = new HashMap<>();
    int left = 0, best = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (last.containsKey(c) && last.get(c) >= left)
            left = last.get(c) + 1;     // shrink
        last.put(c, right);
        best = Math.max(best, right - left + 1);
    }
    return best;
}
PYTHON

Pattern: Expand right every step; shrink left only while a constraint is violated. Keep a running aggregate (count, sum, freq map) so each element enters and leaves the window once → O(n). Works for "longest/shortest subarray with property X."

String algorithms (palindromes & matching)

When & whyA few string-specific techniques show up repeatedly. Expand-around-center solves palindrome problems (longest palindromic substring, count palindromic substrings) in O(n²)/O(1) by trying each of the 2n-1 centers and growing outward. For substring search, KMP finds a pattern in O(n+m) using a prefix-function (failure table) so it never re-examines text characters, and Rabin-Karp uses a rolling hash to compare windows in O(1) — handy for multiple-pattern or "find duplicated substring" problems.

JAVA — expand around center
String longestPalindrome(String s) {
    int start = 0, end = 0;
    for (int i = 0; i < s.length(); i++) {
        int len1 = expand(s, i, i);     // odd length
        int len2 = expand(s, i, i + 1); // even length
        int len = Math.max(len1, len2);
        if (len > end - start + 1) {
            start = i - (len - 1) / 2;
            end = i + len / 2;
        }
    }
    return s.substring(start, end + 1);
}
int expand(String s, int l, int r) {
    while (l >= 0 && r < s.length() && s.charAt(l)==s.charAt(r)) {
        l--; r++;
    }
    return r - l - 1;                    // matched length
}
PYTHON — expand + KMP table
def longest_palindrome(s):
    start = end = 0
    def expand(l, r):
        while l >= 0 and r < len(s) and s[l] == s[r]:
            l -= 1; r += 1
        return l + 1, r - 1              # bounds of palindrome
    for i in range(len(s)):
        for l, r in (expand(i, i), expand(i, i + 1)):
            if r - l > end - start:
                start, end = l, r
    return s[start:end + 1]

# KMP prefix function (failure table):
def kmp_table(p):
    f = [0] * len(p)
    k = 0
    for i in range(1, len(p)):
        while k and p[i] != p[k]: k = f[k-1]
        if p[i] == p[k]: k += 1
        f[i] = k
    return f          # use to match in O(n+m)

Remember: palindrome substring problems → expand around center, and don't forget the even-length centers (between two chars). For pattern matching, Python's in / str.find is usually enough in interviews; reach for KMP only when asked to implement matching or when naive O(n·m) is too slow. Rolling-hash (Rabin-Karp) is the tool for "longest duplicate substring" (binary-search the length + hash).

Monotonic stack / deque

When & whyA monotonic stack keeps elements in increasing or decreasing order, popping ones that violate the order as you go. It answers "next/previous greater or smaller element" in O(n) total — the engine behind daily-temperatures, next-greater-element, largest-rectangle-in-histogram, trapping-rain-water, and sum-of-subarray-minimums. The monotonic deque variant maintains the max/min of a sliding window in O(1) amortized (sliding-window-maximum) by dropping dominated elements from the back and stale indices from the front. Trigger phrases: "next greater," "span," "histogram," "window max/min." The insight: each element is pushed and popped at most once, so the inner while-loop doesn't make it O(n²).

JAVA
// Next Greater Element: for each i, nearest j>i with a[j]>a[i]
int[] nextGreater(int[] a) {
    int n = a.length;
    int[] res = new int[n];
    Arrays.fill(res, -1);
    Deque<Integer> st = new ArrayDeque<>();   // indices, decreasing
    for (int i = 0; i < n; i++) {
        while (!st.isEmpty() && a[st.peek()] < a[i])
            res[st.pop()] = a[i];
        st.push(i);
    }
    return res;
}

// Sliding window maximum (size k) with a deque of indices:

    }
    return res;

Intervals & greedy

When & whyInterval problems almost always start the same way: sort by start (to merge) or by end (to schedule), then sweep once. Merge-intervals, insert-interval, and meeting-rooms (min rooms = max overlap, found with a min-heap of end times or a sweep line) all follow from this. The broader idea is greedy: make the locally optimal choice and prove it's globally optimal — pick the interval that ends earliest to maximize non-overlapping count (activity selection / erase-overlap-intervals), or jump as far as reachable (jump-game). Greedy works when an exchange argument holds; if it doesn't, fall back to DP. Recognize by "intervals," "meetings," "minimum number of …," "maximum non-overlapping."

JAVA
// Merge overlapping intervals:
int[][] merge(int[][] iv) {
    Arrays.sort(iv, (a, b) -> Integer.compare(a[0], b[0]));
    List<int[]> out = new ArrayList<>();
    for (int[] cur : iv) {
        if (out.isEmpty() || out.get(out.size()-1)[1] < cur[0])
            out.add(cur);                       // no overlap
        else
            out.get(out.size()-1)[1] =
                Math.max(out.get(out.size()-1)[1], cur[1]);
    }
    return out.toArray(new int[0][]);
}

// Min meeting rooms = max concurrent intervals:
int minRooms(int[][] iv) {
    int n = iv.length;
    int[] starts = new int[n], ends = new int[n];
    for (int i = 0; i < n; i++) { starts[i]=iv[i][0]; ends[i]=iv[i][1]; }
    Arrays.sort(starts); Arrays.sort(ends);
    int rooms = 0, max = 0, j = 0;
    for (int i = 0; i < n; i++) {
        while (j < n && ends[j] <= starts[i]) { rooms--; j++; }
        rooms++; max = Math.max(max, rooms);
    }
    return max;
}

BFS (shortest path on unweighted graph / grid)

When & whyBFS explores in rings of increasing distance, so it's the go-to for shortest path / fewest steps in an unweighted graph or grid — word ladder, rotting oranges, knight moves, maze escape. The level-snapshot loop (process exactly q.size() nodes per round) is what lets you count distance/steps. It's also used for level-order tree traversal and "multi-source" spread (seed the queue with all sources at once). Use BFS over DFS specifically when you need the shortest path or a layer count; if you only need reachability/connectivity, either works. O(V+E).

JAVA
int bfs(int[][] grid, int sr, int sc) {
    int R = grid.length, C = grid[0].length;
    boolean[][] seen = new boolean[R][C];
    Deque<int[]> q = new ArrayDeque<>();
    q.offer(new int[]{sr, sc});
    seen[sr][sc] = true;
    int dist = 0;
    int[][] DIRS = {{0,1},{0,-1},{1,0},{-1,0}};
    while (!q.isEmpty()) {
        int sz = q.size();          // level boundary
        for (int k = 0; k < sz; k++) {
            int[] cur = q.poll();
            for (int[] d : DIRS) {
                int nr = cur[0]+d[0], nc = cur[1]+d[1];
                if (nr<0||nr>=R||nc<0||nc>=C) continue;
                if (seen[nr][nc]||grid[nr][nc]==1) continue;
                seen[nr][nc] = true;
                q.offer(new int[]{nr, nc});
            }
        }
        dist++;
    }
    return dist;
}

            for dr, dc in DIRS:

                seen[nr][nc] = True
                q.append((nr, nc))
        dist += 1
    return dist

Key idea: Mark visited when enqueueing, not when dequeuing, to avoid adding a node twice. The inner for sz loop snapshots one "level" so dist counts edges/layers — needed for shortest-path and "minimum steps" problems.

DFS (recursive & iterative)

When & whyDFS goes deep before wide, which makes it natural for "explore everything reachable" and bottom-up aggregation: count/flood-fill connected components (number of islands), detect cycles, clone a graph, compute tree heights/sums, and enumerate paths. It uses the call stack (recursive) or an explicit stack (iterative). Prefer DFS when you don't need shortest path and the recursion mirrors the structure (trees, regions); prefer the iterative version when depth can blow the stack. O(V+E). Remember: BFS finds shortest paths, DFS does not — pick by whether distance matters.

JAVA
// recursive grid flood fill / count islands:
void dfs(int[][] g, int r, int c) {

    dfs(g, r, c+1); dfs(g, r, c-1);
}

// iterative with explicit stack (avoids deep recursion):
void dfsIter(List<List<Integer>> adj, int s) {
    boolean[] seen = new boolean[adj.size()];
    Deque<Integer> st = new ArrayDeque<>();
    st.push(s);
    while (!st.isEmpty()) {
        int u = st.pop();
        if (seen[u]) continue;
        seen[u] = true;
        for (int v : adj.get(u))
            if (!seen[v]) st.push(v);
    }

Backtracking (permutations / subsets / combinations)

When & whyBacktracking is brute-force search done cleanly: build a candidate incrementally, recurse, then undo the last choice to try the next. Use it to enumerate all valid configurations — permutations, subsets, combinations, N-Queens, Sudoku, word search, generate-parentheses, combination-sum. The signal is "find/return all" or "count all ways" with a branching decision at each step. It's exponential by nature, so the real craft is pruning early (skip choices that can't lead to a solution) and sorting input to skip duplicates. When you only need the best/count rather than every solution, that's usually DP instead.

JAVA
List<List<Integer>> res = new ArrayList<>();

void backtrack(int[] nums, List<Integer> path,
               boolean[] used) {
    if (path.size() == nums.length) {
        res.add(new ArrayList<>(path));  // COPY!
        return;
    }
    for (int i = 0; i < nums.length; i++) {
        if (used[i]) continue;
        used[i] = true;
        path.add(nums[i]);          // choose
        backtrack(nums, path, used);
        path.remove(path.size()-1); // un-choose
        used[i] = false;
    }
}

Dynamic programming

When & whyDP applies when a problem has overlapping subproblems (the same smaller cases recur) and optimal substructure (the answer is built from answers to those subproblems). Signals: "max/min cost," "number of ways," "can you reach/partition," "longest/shortest subsequence." The workflow that scores points: define the state (what do the indices mean?), write the recurrence (how does a state depend on smaller ones?), set base cases, then choose a direction. Top-down memoization (recursion + cache) is easiest to derive — in Python just slap @lru_cache on the recursion. Bottom-up tabulation avoids recursion limits and enables space optimization (often you only need the previous row/value). Start top-down to get correctness, convert to bottom-up if you need speed/space.

Segment tree & Fenwick (BIT)

When & whyPrefix sums give O(1) range queries but break when the array is updated (every update is O(n) to rebuild). When you need both range queries and point updates in O(log n), use a Fenwick tree (BIT) for sums, or a segment tree for any associative op (min/max/gcd) and range updates. These power "range sum query – mutable," "count of smaller numbers after self," and order-statistics problems.

JAVA — Fenwick / BIT (prefix sums, mutable)
class BIT {                  // 1-indexed
    long[] tree;
    BIT(int n) { tree = new long[n + 1]; }
    void update(int i, long delta) {      // a[i] += delta
        for (; i < tree.length; i += i & (-i)) tree[i] += delta;
    }
    long prefix(int i) {                  // sum a[1..i]
        long s = 0;
        for (; i > 0; i -= i & (-i)) s += tree[i];
        return s;
    }
    long range(int l, int r) {            // sum a[l..r]
        return prefix(r) - prefix(l - 1);
    }
}
// i & (-i) isolates the lowest set bit = the node's span
PYTHON — Fenwick / BIT

Quickselect (k-th smallest / largest)

When & whyQuickselect finds the k-th smallest/largest element in average O(n) — better than sorting (O(n log n)) or a heap (O(n log k)) — by partitioning like quicksort but only recursing into the side that contains k. Use it for "k-th largest element," "top-k" when you can rearrange the array, and median-of-medians style problems. The trade-off: worst case is O(n²) on adversarial input (mitigate with a random pivot), and it reorders the array. If you can't mutate the input or need a streaming solution, use a heap instead.

        else hi = p - 1;

Union-Find (Disjoint Set Union)

When & whyUnion-Find answers "are these two things in the same group?" and "merge these two groups" in near-O(1) amortized time. Reach for it on connectivity problems where edges arrive incrementally and you don't need actual paths: number of provinces/friend-circles, redundant-connection (cycle detection in an undirected graph), accounts-merge, and Kruskal's MST. It's often simpler and faster than BFS/DFS when the question is purely "how many components" or "do these connect." Choose DFS/BFS instead when you need the path itself or distances.

JAVA
class DSU {
    int[] parent, rank;
    DSU(int n) {
        parent = new int[n]; rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }
    int find(int x) {
        while (parent[x] != x) {
            parent[x] = parent[parent[x]]; // path compress
            x = parent[x];
        }
        return x;
    }
    boolean union(int a, int b) {
        int ra = find(a), rb = find(b);
        if (ra == rb) return false;       // already joined
        if (rank[ra] < rank[rb]) { int t=ra; ra=rb; rb=t; }
        parent[rb] = ra;
        if (rank[ra] == rank[rb]) rank[ra]++;
        return true;
    }
}

        self.rank = [0] * n

            self.parent[x] = self.parent[self.parent[x]]

        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt
    return prev

# dummy head for clean insert/delete:
dummy = ListNode(0)
dummy.next = head
# fast/slow to find middle / detect cycle:
slow = fast = head
while fast and fast.next:
    slow = slow.next
    fast = fast.next.next

Tip: Two patterns crack most linked-list problems: a dummy head (avoids special-casing the first node on insert/delete) and the fast/slow pointers (find middle, detect cycles via Floyd, find n-th from end). Always save cur.next before you rewire it.

Binary tree traversal

When & whyAlmost every tree problem is one of four traversals with a tweak. Inorder (left-node-right) yields a BST's values in sorted order — use it for validate-BST and k-th smallest. Postorder (left-right-node) is for bottom-up aggregation where a node needs its children's results first — height, diameter, "is balanced," subtree sums. Preorder (node-left-right) suits top-down passes and serialization. Level-order (BFS) handles "by depth" questions — right-side view, level averages, zigzag. Pick the order by asking: do I need data from my children before I can compute my answer (postorder) or am I passing context down (preorder)?

JAVA
class TreeNode {
    int val; TreeNode left, right;
    TreeNode(int v) { val = v; }
}

    inorder(n.left, out);
    out.add(n.val);
    inorder(n.right, out);

Trie (prefix tree)

When & whyA trie stores a set of strings by shared prefix, so prefix queries cost O(length) regardless of how many words you've inserted — far better than scanning every word. Use it when the problem is explicitly about prefixes or a dictionary of words: implement-trie, word-search-II (trie + grid DFS to prune dead branches early), autocomplete/search-suggestions, replace-words, and maximum-XOR-of-two-numbers (a binary trie over bits). If you only ever check whole-word membership, a hash set is simpler; the trie earns its keep the moment prefixes or shared structure matter.

JAVA
class Trie {
    Trie[] kids = new Trie[26];
    boolean end = false;

        Trie cur = this;

            int i = c - 'a';

        Trie cur = this;
        for (char c : s.toCharArray()) {

            if (cur == null) return null;

        return cur;
    }
}

Dijkstra (shortest path, weighted)

When & whyDijkstra is BFS for weighted graphs with non-negative edges: instead of a plain queue it uses a min-heap to always expand the closest-so-far node, giving shortest paths in O(E log V). Use it for network-delay-time, cheapest-flights (with a hop limit), path-with-minimum-effort, and swim-in-rising-water. The required habit is the stale-entry skip (if d > dist[u]: continue) because we push improved distances rather than decrease keys in place. If edges can be negative, Dijkstra breaks — use Bellman-Ford; if the graph is unweighted, plain BFS is simpler and faster.

JAVA
int[] dijkstra(List<int[]>[] adj, int src, int n) {
    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[src] = 0;
    // min-heap of {dist, node}:
    PriorityQueue<int[]> pq =
        new PriorityQueue<>((a,b) -> a[0] - b[0]);
    pq.offer(new int[]{0, src});
    while (!pq.isEmpty()) {
        int[] top = pq.poll();
        int d = top[0], u = top[1];
        if (d > dist[u]) continue;       // stale entry
        for (int[] e : adj[u]) {          // e = {v, w}
            int v = e[0], w = e[1];
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.offer(new int[]{dist[v], v});
            }
        }
    }
    return dist;
}
PYTHON

    return dist

Tip: Always skip stale heap entries with the if d > dist[u]: continue check — Dijkstra never decreases keys in place, it just pushes new better ones. Python's tuple ordering makes (dist, node) sort correctly for free. Dijkstra requires non-negative weights; with negatives use Bellman-Ford.

Topological sort (Kahn's BFS)

When & whyTopological sort linearizes a directed acyclic graph so that every edge points "forward" — exactly what you need for ordering by dependency: course-schedule (can you finish? in what order?), build systems, task scheduling with prerequisites, alien-dictionary. Kahn's BFS version (repeatedly remove zero-indegree nodes) doubles as cycle detection: if you can't output all n nodes, a cycle exists and no valid order does. Use it whenever the problem mentions "prerequisites," "depends on," or "must come before." O(V+E).

JAVA
// returns order, or empty list if a cycle exists:
List<Integer> topo(List<List<Integer>> adj, int n) {
    int[] indeg = new int[n];
    for (List<Integer> nbrs : adj)
        for (int v : nbrs) indeg[v]++;
    Deque<Integer> q = new ArrayDeque<>();
    for (int i = 0; i < n; i++)
        if (indeg[i] == 0) q.offer(i);
    List<Integer> order = new ArrayList<>();
    while (!q.isEmpty()) {
        int u = q.poll();
        order.add(u);
        for (int v : adj.get(u))
            if (--indeg[v] == 0) q.offer(v);
    }
    return order.size() == n ? order
                             : new ArrayList<>();
}
PYTHON
from collections import deque

def topo(adj, n):
    indeg = [0] * n

        for v in nbrs:
            indeg[v] += 1
    q = deque(i for i in range(n) if indeg[i] == 0)

        u = q.popleft()
        order.append(u)

            if indeg[v] == 0:

    return order if len(order) == n else []  # [] = cycle

Two Sum  ● Easy · hash map

Given an array nums and a target, return indices of the two numbers that add up to target. Exactly one solution; can't reuse an element.

ApproachBrute force checks every pair — O(n²). Instead, make one pass with a hash map of value → index. For each number x, its partner is target - x; if you've already seen the partner, you're done. You trade O(n) space for O(n) time. Time O(n), Space O(n).

JAVA
int[] twoSum(int[] nums, int target) {
    Map<Integer,Integer> seen = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int need = target - nums[i];
        if (seen.containsKey(need))
            return new int[]{seen.get(need), i};
        seen.put(nums[i], i);          // store AFTER check
    }
    return new int[]{};                // no solution
}
PYTHON
def two_sum(nums, target):
    seen = {}                          # value -> index
    for i, x in enumerate(nums):
        need = target - x
        if need in seen:
            return [seen[need], i]
        seen[x] = i                    # store AFTER check
    return []

Key insight: "find a pair that sums to k" → one-pass hash map of complements. Storing after the check prevents using the same element twice. This complement trick generalizes to 3Sum (sort + fix one + two-pointer).

Valid Anagram  ● Easy · counting

Return true if string t is an anagram of s (same characters, same counts).

ApproachTwo strings are anagrams iff their character-frequency tables are equal. For lowercase letters, an int[26] is fastest: +1 for each char in s, −1 for each in t; all zero ⇒ anagram. (Sorting both and comparing also works but is O(n log n).) Time O(n), Space O(1) (fixed 26).

JAVA
boolean isAnagram(String s, String t) {
    if (s.length() != t.length()) return false;
    int[] cnt = new int[26];
    for (int i = 0; i < s.length(); i++) {
        cnt[s.charAt(i) - 'a']++;
        cnt[t.charAt(i) - 'a']--;
    }
    for (int c : cnt) if (c != 0) return false;
    return true;
}
PYTHON

Best Time to Buy and Sell Stock  ● Easy · one pass

One transaction (buy once, sell later). Maximize profit from prices[]; return 0 if no profit.

ApproachTrack the minimum price seen so far as you sweep left to right; at each day, the best profit selling today is price - minSoFar. Keep the max of those. You never need the future — the lowest buy is always behind you. Time O(n), Space O(1).

JAVA
int maxProfit(int[] prices) {
    int minPrice = Integer.MAX_VALUE, best = 0;
    for (int p : prices) {
        minPrice = Math.min(minPrice, p);   // best buy so far
        best = Math.max(best, p - minPrice); // sell today
    }
    return best;
}

Valid Parentheses  ● Easy · stack

Given a string of ()[]{}, decide if every bracket is properly closed and nested.

ApproachNesting = "last opened, first closed" = a stack. Push opening brackets; on a closing bracket, the top of the stack must be its matching opener — otherwise invalid. At the end the stack must be empty. Time O(n), Space O(n).

JAVA
boolean isValid(String s) {
    Deque<Character> st = new ArrayDeque<>();
    Map<Character,Character> pair = Map.of(')','(', ']','[', '}','{');
    for (char c : s.toCharArray()) {
        if (pair.containsKey(c)) {               // closing
            if (st.isEmpty() || st.pop() != pair.get(c))
                return false;
        } else st.push(c);                        // opening
    }
    return st.isEmpty();
}
PYTHON

Merge Two Sorted Lists  ● Easy · linked list

Merge two sorted linked lists into one sorted list and return its head.

ApproachUse a dummy head to avoid special-casing the first node, and a tail pointer you keep appending to. Walk both lists, always attaching the smaller current node and advancing that list; finally attach whatever remains. Time O(n+m), Space O(1).

JAVA
ListNode merge(ListNode a, ListNode b) {
    ListNode dummy = new ListNode(0), tail = dummy;
    while (a != null && b != null) {
        if (a.val <= b.val) { tail.next = a; a = a.next; }
        else                { tail.next = b; b = b.next; }
        tail = tail.next;
    }
    tail.next = (a != null) ? a : b;     // attach the rest
    return dummy.next;
}
PYTHON

Maximum Subarray (Kadane)  ● Medium · DP

Find the contiguous subarray with the largest sum; return that sum.

ApproachKadane's: at each index decide whether to extend the running subarray or start fresh at the current element — cur = max(x, cur + x). Track the best cur ever seen. Intuition: a negative running sum can only hurt what follows, so drop it. Time O(n), Space O(1).

JAVA
int maxSubArray(int[] nums) {
    int cur = nums[0], best = nums[0];
    for (int i = 1; i < nums.length; i++) {
        cur = Math.max(nums[i], cur + nums[i]); // extend or restart
        best = Math.max(best, cur);
    }
    return best;
}
PYTHON

Product of Array Except Self  ● Medium · prefix

Return out[i] = product of all elements except nums[i], without division and in O(n).

Approachout[i] = (product of everything to the left) × (everything to the right). Do a left-to-right pass filling prefix products, then a right-to-left pass multiplying in the suffix product on the fly. No division (handles zeros), O(1) extra space beyond the output. Time O(n), Space O(1).

JAVA
int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] out = new int[n];
    out[0] = 1;
    for (int i = 1; i < n; i++)           // prefix products
        out[i] = out[i-1] * nums[i-1];
    int right = 1;
    for (int i = n-1; i >= 0; i--) {       // multiply suffix
        out[i] *= right;
        right *= nums[i];
    }
    return out;
}
PYTHON

Longest Substring Without Repeating Characters  ● Medium · sliding window

Length of the longest substring with all distinct characters.

ApproachSliding window with a map of each char's last index. Expand right; when you hit a char already inside the window, jump left past its previous position. The window [left, right] always holds distinct chars; track its max length. Each char is visited once. Time O(n), Space O(min(n, alphabet)).

JAVA
int lengthOfLongestSubstring(String s) {
    Map<Character,Integer> last = new HashMap<>();
    int left = 0, best = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (last.containsKey(c) && last.get(c) >= left)
            left = last.get(c) + 1;        // shrink past dup
        last.put(c, right);
        best = Math.max(best, right - left + 1);
    }
    return best;
}
PYTHON

Top K Frequent Elements  ● Medium · heap / bucket

Return the k most frequent elements in nums.

ApproachCount frequencies, then pick the top k. A size-k min-heap of (freq, value) gives O(n log k). Even better, bucket sort by frequency (index = count, since count ≤ n) gives O(n). Time O(n) bucket / O(n log k) heap.

JAVA
int[] topKFrequent(int[] nums, int k) {
    Map<Integer,Integer> cnt = new HashMap<>();
    for (int x : nums) cnt.merge(x, 1, Integer::sum);
    // min-heap of size k by frequency
    PriorityQueue<int[]> pq =
        new PriorityQueue<>((a,b) -> a[1] - b[1]);
    for (var e : cnt.entrySet()) {
        pq.offer(new int[]{e.getKey(), e.getValue()});
        if (pq.size() > k) pq.poll();          // drop smallest
    }
    int[] res = new int[k];
    for (int i = 0; i < k; i++) res[i] = pq.poll()[0];
    return res;
}

    buckets = [[] for _ in range(len(nums) + 1)]
    for v, c in cnt.items():
        buckets[c].append(v)
    res = []
    for c in range(len(buckets) - 1, 0, -1):
        res += buckets[c]
        if len(res) >= k: return res[:k]

Key insight: "top K" → size-K heap (keep the heap small, not size-n). When the score is a bounded integer (a frequency ≤ n), bucket sort by that score beats the heap at O(n).

Course Schedule  ● Medium · topological sort

Given numCourses and prerequisite pairs [a, b] (b before a), can you finish all courses?

ApproachThis is "does a directed graph have a cycle?" Build the graph, compute in-degrees, and run Kahn's BFS: repeatedly remove zero-in-degree nodes. If you process all N nodes, no cycle → finishable; if fewer, a cycle blocks it. Time O(V+E), Space O(V+E).

JAVA
boolean canFinish(int n, int[][] prereq) {
    List<List<Integer>> adj = new ArrayList<>();
    for (int i = 0; i < n; i++) adj.add(new ArrayList<>());
    int[] indeg = new int[n];
    for (int[] p : prereq) { adj.get(p[1]).add(p[0]); indeg[p[0]]++; }
    Deque<Integer> q = new ArrayDeque<>();
    for (int i = 0; i < n; i++) if (indeg[i]==0) q.offer(i);
    int done = 0;
    while (!q.isEmpty()) {
        int u = q.poll(); done++;
        for (int v : adj.get(u)) if (--indeg[v]==0) q.offer(v);
    }
    return done == n;                    // all processed = no cycle
}
PYTHON
from collections import deque

Coin Change  ● Medium · DP

Fewest coins from coins[] to make amount; return −1 if impossible.

ApproachUnbounded-knapsack DP. dp[x] = min coins to make x. For each amount x, try every coin: dp[x] = min(dp[x], dp[x-coin] + 1). Initialize to "infinity," dp[0]=0. Greedy fails (e.g. coins {1,3,4}, amount 6), so DP is required. Time O(amount × coins), Space O(amount).

JAVA
int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1);          // "infinity"
    dp[0] = 0;
    for (int x = 1; x <= amount; x++)
        for (int c : coins)
            if (c <= x) dp[x] = Math.min(dp[x], dp[x-c] + 1);
    return dp[amount] > amount ? -1 : dp[amount];
}
PYTHON

Merge Intervals  ● Medium · sort + greedy

Merge all overlapping intervals in [[start,end],…].

ApproachSort by start. Sweep once: if the current interval overlaps the last one in the output (cur.start ≤ last.end), extend last.end; otherwise append a new interval. Sorting makes overlaps adjacent, so one pass suffices. Time O(n log n), Space O(n).

JAVA
int[][] merge(int[][] iv) {

    List<int[]> out = new ArrayList<>();
    for (int[] cur : iv) {
        if (out.isEmpty() || out.get(out.size()-1)[1] < cur[0])
            out.add(cur);                          // no overlap
        else
            out.get(out.size()-1)[1] =
                Math.max(out.get(out.size()-1)[1], cur[1]);
    }
    return out.toArray(new int[0][]);
}
PYTHON

LRU Cache  ● Medium · design (hashmap + DLL)

Design a cache with O(1) get and put that evicts the least-recently-used key at capacity.

ApproachCombine a hash map (key → node, O(1) lookup) with a doubly linked list ordered by recency (most-recent at the head, LRU at the tail). Every access moves the node to the head; eviction removes the tail. Time O(1) per op.

JAVA
class LRUCache extends LinkedHashMap<Integer,Integer> {
    private final int cap;
    LRUCache(int capacity) {
        super(capacity, 0.75f, true);     // accessOrder = true
        this.cap = capacity;
    }
    public int get(int key) { return super.getOrDefault(key, -1); }
    public void put(int key, int val) { super.put(key, val); }
    protected boolean removeEldestEntry(Map.Entry<Integer,Integer> e) {
        return size() > cap;              // auto-evict LRU
    }
}
// Interview-preferred: hand-roll HashMap + doubly linked list.
PYTHON

    def __init__(self, capacity):
        self.cap = capacity
        self.d = OrderedDict()            # insertion = recency

        self.d.move_to_end(key)           # mark most-recent

Trapping Rain Water  ● Hard · two pointers

Given bar heights, compute how much water is trapped after raining.

ApproachWater above bar i = min(maxLeft, maxRight) - height[i]. The O(n)/O(1) trick: two pointers from both ends moving inward, advancing the smaller side — because that side's running max is the true bound. Track leftMax/rightMax and add the gap. Time O(n), Space O(1).

JAVA
int trap(int[] h) {
    int l = 0, r = h.length - 1, lm = 0, rm = 0, water = 0;
    while (l < r) {
        if (h[l] < h[r]) {                // left side bounds
            lm = Math.max(lm, h[l]);
            water += lm - h[l];
            l++;
        } else {                          // right side bounds
            rm = Math.max(rm, h[r]);
            water += rm - h[r];
            r--;
        }
    }
    return water;
}
PYTHON

3Sum  ● Medium · sort + two pointers

Find all unique triplets [a,b,c] in nums with a+b+c = 0.

ApproachSort the array. Fix the first element i, then two-pointer the rest for pairs summing to -nums[i]. Sorting lets you move pointers by comparison and, crucially, skip duplicates to avoid repeat triplets. Time O(n²), Space O(1) (besides output).

JAVA
List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums);
    List<List<Integer>> res = new ArrayList<>();
    for (int i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] == nums[i-1]) continue;  // skip dup
        int l = i + 1, r = nums.length - 1;
        while (l < r) {
            int sum = nums[i] + nums[l] + nums[r];
            if (sum == 0) {
                res.add(Arrays.asList(nums[i], nums[l], nums[r]));
                while (l < r && nums[l]==nums[l+1]) l++;
                while (l < r && nums[r]==nums[r-1]) r--;
                l++; r--;
            } else if (sum < 0) l++;
            else r--;
        }
    }
    return res;
}
PYTHON

Search in Rotated Sorted Array  ● Medium · binary search

A sorted array rotated at an unknown pivot; find target's index in O(log n) or −1.

ApproachBinary search still works because at any midpoint, at least one half is sorted. Check which half is sorted (compare nums[lo] to nums[mid]), test whether the target lies within that sorted half, and discard the other half. Time O(log n), Space O(1).

JAVA
int search(int[] a, int target) {
    int lo = 0, hi = a.length - 1;
    while (lo <= hi) {
        int mid = (lo + hi) >>> 1;
        if (a[mid] == target) return mid;
        if (a[lo] <= a[mid]) {            // left half sorted
            if (a[lo] <= target && target < a[mid]) hi = mid - 1;
            else lo = mid + 1;
        } else {                          // right half sorted
            if (a[mid] < target && target <= a[hi]) lo = mid + 1;
            else hi = mid - 1;
        }
    }
    return -1;
}
PYTHON

Binary Tree Level Order Traversal  ● Medium · BFS

Return node values grouped by level (top to bottom).

ApproachBFS with a queue, processing one level at a time: snapshot the queue size at the start of each round so you know exactly how many nodes belong to the current level. Time O(n), Space O(n).

JAVA
List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Deque<TreeNode> q = new ArrayDeque<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int sz = q.size();               // nodes in this level
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < sz; i++) {
            TreeNode n = q.poll();
            level.add(n.val);
            if (n.left != null)  q.offer(n.left);
            if (n.right != null) q.offer(n.right);
        }
        res.add(level);
    }
    return res;
}

Clone Graph  ● Medium · graph traversal + map

ApproachTraverse (DFS or BFS) while keeping a map from original node → its clone. The map both prevents infinite loops on cycles and lets you wire cloned neighbors. Create a clone on first visit, then recurse/enqueue neighbors. Time O(V+E), Space O(V).

JAVA

House Robber  ● Medium · DP

Max sum of non-adjacent elements (can't rob two adjacent houses).

ApproachAt each house, choose: skip it (keep best so far) or rob it (its value + best up to two houses back). dp[i] = max(dp[i-1], dp[i-2] + nums[i]). Only the last two values are needed → O(1) space. Time O(n), Space O(1).

JAVA
int rob(int[] nums) {
    int prev2 = 0, prev1 = 0;             // dp[i-2], dp[i-1]
    for (int x : nums) {
        int cur = Math.max(prev1, prev2 + x); // skip vs rob
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
PYTHON

Min Stack  ● Easy · design

Stack supporting push, pop, top, and getMin — all in O(1).

ApproachKeep a second stack of the running minimum that moves in lockstep: each entry records the min of the main stack up to that point. Pushing stores min(x, currentMin); popping removes from both. getMin reads the top of the min stack. Time O(1) per op, Space O(n).

JAVA
class MinStack {
    Deque<Integer> st = new ArrayDeque<>();
    Deque<Integer> mins = new ArrayDeque<>();
    public void push(int x) {
        st.push(x);
        mins.push(mins.isEmpty() ? x : Math.min(x, mins.peek()));
    }
    public void pop()    { st.pop(); mins.pop(); }
    public int top()     { return st.peek(); }
    public int getMin()  { return mins.peek(); }
}
PYTHON
